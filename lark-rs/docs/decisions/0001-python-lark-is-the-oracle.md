# ADR-0001: Python Lark is the oracle (oracle-first testing)

- **Status:** Accepted
- **Date:** backfilled 2026-06-13 (decision predates the record; set at project start)

## Context

We are reimplementing a mature, behavior-rich parser. Parsers are hard to write
correctly but easy to *verify*: given an input, there is one right tree. Python
Lark already defines what "right" means for every grammar feature we care about.
We needed a way to test that didn't depend on us hand-authoring thousands of
expected trees (which would itself be error-prone and would encode our own
misunderstandings).

> "Traditional computers automate what you can specify in code. AI/LLMs automate
> what you can verify." — Andrej Karpathy

## Decision

Python Lark is the ground-truth **oracle**. We generate expected parse trees by
running Python Lark and compare lark-rs's output against them. Concretely:

- Every new grammar feature gets an oracle test *before* it is implemented.
- Every bug is first reproduced as a failing oracle/test, then fixed.
- A suspected performance pathology must be a committed, deterministic scaling
  benchmark before it is fixed (see [ADR-0007](0007-deterministic-perf-counters.md)).
- Prefer end-to-end oracle tests over unit tests — the oracle checks the whole
  pipeline.

Oracle JSON is generated by `tools/generate_oracles.py` and committed so tests
run without Python. Generated fixtures are never hand-edited
(`.claude/settings.json` denies it); regenerate via the tools.

## Consequences

- Correctness questions reduce to "is there an oracle/compliance case that pins
  this?" — the test suite is the real spec, not the prose docs.
- We inherit Lark's behavior *including* its quirks; where the regex dialects
  genuinely differ we normalize toward Python (see
  [ADR-0004](0004-python-re-regex-dialect.md)).
- One sharp edge: we never write an oracle that depends on an arbitrary lexer
  tie-break (two terminals matching the same span at equal priority, which Lark
  resolves by an incidental regex-source-length sort we don't reproduce).
  Disambiguate with explicit terminal priority instead, exactly as the Lark
  authors do.
- Enforced by `tests/test_oracle_coverage.rs` (every grammar needs an oracle or
  a quarantine entry) and a CI freshness gate that regenerates the oracles and
  fails on drift.
