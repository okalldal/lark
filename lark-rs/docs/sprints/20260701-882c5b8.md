(#582): the ledger + omnibus body claimed "no new public API (internal `TokenValueMode` enum only)",
but the integrated code exposed it as **public API** — `pub enum TokenValueMode`, `BasicLexer::lex_span`,
`ContextualLexer::next_token_with_mode` (takes `TokenValueMode`), and `ContextualLexer::next_token_span`
were all `pub` inside the `pub mod lexer` (with `BasicLexer`/`ContextualLexer` re-exported at the crate
root). The "tier auto, no public API" argument was therefore false as staged.

**Verified and agreed.** All four symbols are used **only within the `lark-rs` crate**
(`parsers/mod.rs`, `parsers/token_source.rs`) — no binding/standalone/bench/example usage — so the fix
is a pure visibility narrowing to `pub(crate)`. Applied as commit `e459d70` **directly on the sprint
integration branch** (the orchestrator owns it; the normal follow-up-child-PR route à la #585→#586 was
unavailable because the GitHub MCP server disconnected mid-session). Verified: builds clean under
`--features "span-tree perf-counters"`, `test_span_tree` 9/9, fast gate (fmt + `cargo test --all`) green.
This restores the "internal perf change, no public API" contract that makes #582 `auto`-tier — the
omnibus body's claim is now accurate, no retraction needed.
